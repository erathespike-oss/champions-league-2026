# 🏆 Champions League 2026 Analytics Dashboard

Профессиональный аналитический дашборд для Лиги Чемпионов 2026. Проект перешел от математических симуляций к работе с реальными историческими данными (UCL 24/25) через Django ORM и кастомный ETL-пайплайн.

## 🚀 Быстрый старт (для разработчиков)

1. **Клонирование проекта:**
   ```bash
   git clone https://github.com/erathespike-oss/champions-league-2026.git
   cd champions-league-2026
   ```

2. **Настройка окружения:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. **База данных и данные:**
   ```bash
   python manage.py migrate
   # Импорт реальных данных из CSV (убедитесь, что файл в data/)
   python manage.py shell -c "from dashboard.core.etl import KaggleETL; KaggleETL().run_pipeline('data/football_matches_2024_2025.csv')"
   ```

4. **Запуск:**
   ```bash
   python manage.py runserver
   ```

## 🛠 Архитектура проекта

- **`dashboard/models.py`**: ORM модели `Team` (статистика, очки, xG) и `Match` (результаты, раунды, видео).
- **`dashboard/core/etl.py`**: Высокопроизводительный загрузчик данных. Использует `csv.DictReader` и `bulk_create` для экономии памяти.
- **`dashboard/views.py`**: 
    - `index`: Рендерит главную таблицу.
    - `team_details`: JSON API для получения истории матчей и статистики конкретной команды.
    - `match_detail`: Страница с глубокой аналитикой конкретной игры.
- **`dashboard/templates/`**: Фронтенд на Tailwind CSS. Вся интерактивность (табы, подгрузка данных) реализована на Vanilla JS.

## 🤝 Как мы работаем (Workflow)

Чтобы мы не мешали друг другу, соблюдаем следующие правила:

1. **Ветки (Branches):**
   - Не пушьте напрямую в `main`.
   - Создавайте ветку под задачу: `git checkout -b feature/h2h-logic` или `git checkout -b fix/table-sorting`.

2. **Коммиты (Commits):**
   - Пишите понятные сообщения на русском или английском.
   - Пример: `git commit -m "Добавлена логика Head-to-Head во вкладку аналитики"`

3. **Загрузка данных:**
   - Если вы меняете структуру моделей, обязательно делайте `python manage.py makemigrations`.
   - Файлы `.sqlite3` и `.csv` игнорируются гитом. Каждый разработчик генерирует свою локальную БД через ETL-скрипт.
